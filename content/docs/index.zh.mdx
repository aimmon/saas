## 前言
Hi, 我是 Jayden，一名全栈开发工程师。过去一段时间里，我和伙伴Evan为 Viber 们打造了 VibeAny，一个面向生产的全栈脚手架。我们观察到，很多人在 vibe coding 新产品时，不得不反复书写样板代码、重复搭建基础设施。VibeAny 改变这一切：认证、支付、AI、国际化等能力开箱即用，让你把时间花在产品本身，从想法到上线更快。


## 为什么选择 vibe-any

下面快速浏览 VibeAny 的核心能力。

### 开箱即用

你只需要花 10-20 分钟的时间配置，即可获得以下所有的服务。

- 自带丰富的 landing page 组件，帮助你快速搭建企业级 landing page，支持响应式，并且 seo 友好。
- 告别繁多的环境变量配置，支持后台动态配置
- 完整认证体系：邮箱密码、Google/GitHub 登录、邮箱验证。
- 内置 AI 能力：文本/图像/视频/音频生成。
- 支付系统：可选 Stripe 、 Creem 、 Paypal、 微信和支付宝等众多支付平台，支持订阅与一次性付款。
- 积分系统：快速落地商业化。
- 管理后台：用户管理开箱即用。
- 数据埋点分析：支持 Google Analytics, Sentry, Posthog 等众多服务商。
- 多语言支持，且类型安全（对比 next-intl）。

## 适用场景

### AI 应用开发

<Callout type="warn" title="痛点">
接入多个 AI Provider（OpenAI、Claude、Gemini、DeepSeek 等）需要分别对接不同的 SDK 和 API 格式；AI 能力的计费管理复杂，难以追踪用户消耗和成本。
</Callout>

VibeAny 通过 Vercel AI SDK 统一抽象多 Provider 调用，支持流式响应、文本/图像/视频/音频生成。内置积分系统采用 FIFO 消费策略，支持每日奖励、过期管理，让你轻松落地 AI 能力的商业化。

### 商业化产品

<Callout type="warn" title="痛点">
从零搭建一个 SaaS 产品，需要自己实现用户认证、订阅管理、支付对接、管理后台，这些"基础设施"往往要花费数周时间。
</Callout>

VibeAny 提供完整的 SaaS 基础设施：Better Auth 驱动的认证系统（邮箱/Google/GitHub 登录）、多支付平台支持（Stripe/Creem/PayPal/微信/支付宝）、订阅与一次性付费方案、用户管理后台，让你跳过重复劳动，专注产品核心功能。

### 出海业务

<Callout type="warn" title="痛点">
国际化配置繁琐，传统方案（如 next-intl）缺乏类型安全，翻译 key 拼写错误只能在运行时发现；海外支付渠道对接需要理解各平台的 Webhook 机制和状态管理。
</Callout>

VibeAny 采用 Intlayer 实现类型安全的国际化，编译时即可捕获翻译 key 错误。支付系统抽象了 Stripe、Creem 等平台的差异，统一处理 Webhook 和订单状态，让出海产品开发更加顺畅。

### 内容网站

<Callout type="warn" title="痛点">
搭建文档或博客系统，需要自己配置 MDX 解析、目录生成、搜索功能、多语言路由，往往需要大量的胶水代码。
</Callout>

VibeAny 集成 Fumadocs，MDX 内容系统开箱即用，自动生成文档导航、支持全文搜索、完美适配多语言路由。无论是产品文档、技术博客还是知识库，都能快速上线。
